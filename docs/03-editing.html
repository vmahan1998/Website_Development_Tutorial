4  Editing Your Content

Personalizing your homepage with text, images, and a live calendar

4.1 Overview

Now that you understand how the files are organized, it’s time to make the website yours. This chapter focuses on editing index.qmd — your homepage — which is the first thing visitors will see.

You’ll learn how to:

  • Replace the placeholder text with your own name and bio
  • Add your own profile photo
  • Arrange text and images side by side using columns
  • Embed a live public Google Calendar
Tip

Before making any edits, it’s a good habit to run quarto preview in the Terminal so the preview is live in your browser. That way you can see your changes reflected in real time as you save the file.

4.2 Step 1: Open Your Homepage File

In RStudio, find index.qmd in the Files panel (bottom-right) and click on it to open it in the editor.

At the very top of the file you’ll see the YAML front matter — a short block between two --- lines that sets page-level options:

---
title: "Vanessa Quintana"
subtitle: "Applied Ecological Modeler • Coastal Systems Scientist • Geospatial Analyst"
page-layout: full
image: images/Author_Picture.jpg
format:
  html:
    toc: false
    smooth-scroll: true
---

Start by updating these fields:

  • title — Replace "Vanessa Quintana" with your own name
  • subtitle — Replace the subtitle with your own roles or research focus areas, separated by • if you’d like the same style
  • image — This points to your profile photo (more on this in Step 2)
Note

The page-layout: full setting gives the homepage a wider layout than the other pages, which is why it looks different from research.qmd or cv.qmd. Leave this as-is for the homepage.

4.3 Step 2: Add Your Profile Photo

Your profile photo is displayed on the left side of the homepage. To replace the placeholder with your own image:

  1. Find a photo you’d like to use — a headshot or professional photo works best
  2. Rename the file to something simple with no spaces, for example my_photo.jpg
  3. Move or copy the file into the images/ folder inside your project
  4. In index.qmd, find the line that references the image:
![](images/Author_Picture.jpg){.profile-image width="100%" fig-alt="Vanessa Quintana"}
  1. Replace Author_Picture.jpg with your own filename, and update the fig-alt text with your name:
![](images/my_photo.jpg){.profile-image width="100%" fig-alt="Your Name"}
Tip

The fig-alt text is the alt text for your image — a short description that appears if the image fails to load and that screen readers use for accessibility. Always include a brief, accurate description.

Important

Make sure your image file is actually inside the images/ folder and that the filename in your .qmd file matches exactly — including capitalization and the file extension (.jpg, .png, etc.). A mismatch here is one of the most common beginner errors.

4.4 Step 3: Edit Your Bio Text

Below the YAML front matter, the homepage is built using columns — a Quarto layout feature that lets you place content side by side. Here’s how the two-column layout is structured in index.qmd:

::: {.columns}
::: {.column width="32%"}

  [your photo and profile panel go here]

:::
::: {.column width="68%"}

  [your bio text goes here]

:::
:::

The first column (32% wide) holds your photo and badge list. The second column (68% wide) holds your bio and research philosophy. To update your bio, find the text in the wider column and replace it with your own:

::: {.column width="68%"}

::: {.kicker}
About My Work
:::

I work at the intersection of [your research areas here]. My research focuses on
[what you study and why it matters].

I approach [your methods or philosophy here].

:::
Note

What is .kicker? The .kicker is a small styled label that appears above a section heading — like a category tag. It’s defined in styles.css and gives the page a clean, editorial look. You can change the text inside it to anything you like, or remove it entirely if you prefer.

4.5 Step 4: Understanding Side-by-Side Layouts (Columns)

The column layout you just saw is one of the most useful features in Quarto for building a polished homepage. Here’s the full pattern so you can use it anywhere on your site:

::: {.columns}
::: {.column width="50%"}

Content for the left side goes here.

You can put text, images, lists, or anything else.

:::
::: {.column width="50%"}

Content for the right side goes here.

:::
:::

The widths must add up to 100%. You can split them however you like — for example 30% / 70%, or 25% / 25% / 25% / 25% for four equal columns (as used in the Research Areas section of the homepage).

4.5.1 Placing an Image Next to Text

A common use of the column layout is putting a photo or figure next to a block of text. Here’s a simple example you can copy and adapt:

::: {.columns}
::: {.column width="40%"}

![](images/my_photo.jpg){width="100%" fig-alt="A description of the image"}

:::
::: {.column width="60%"}

### A heading for the text side

Write your paragraph text here. This will appear to the right of the image.
You can include as much or as little text as you need.

:::
:::
Tip

Add a blank line between the ::: dividers and your content. Quarto needs that spacing to correctly parse the column layout — missing blank lines is a common cause of formatting issues.

4.6 Step 5: Update Your Badge List

In the left column of the homepage, below your photo, there is a badge list — a set of small styled labels showing your affiliation, title, or research focus areas. Find this block in index.qmd:

::: {.badge-list}
Marine Sciences, University of Maine

ORISE Fellow, U.S. Army Corps of Engineers

Estuarine Hydrodynamics and Sediment Transport

Agent-Based and Ecological Modeling
:::

Replace each line with your own affiliations, department, institution, or keywords. Each separate line becomes its own badge. You can add or remove lines as needed.

4.7 Step 6: Embed a Public Google Calendar

If you have a public Google Calendar — for office hours, events, lab meetings, or workshops — you can embed it directly into your website as a live, interactive calendar. Any updates you make in Google Calendar will automatically appear on your site.

4.7.1 First: Make Your Google Calendar Public

  1. Go to calendar.google.com and sign in
  2. In the left panel, find the calendar you want to embed, click the three dots next to its name, and select “Settings and sharing”
  3. Under “Access permissions for events”, check the box that says “Make available to public”
  4. Confirm the warning if prompted

4.7.2 Second: Get the Embed Code

  1. Still on the calendar settings page, scroll down to “Integrate calendar”
  2. Copy the “Public URL to this calendar” — it will look like a long link ending in @group.calendar.google.com
  3. In the same section, click “Customize” to open the embed builder if you’d like to adjust the calendar’s appearance, then copy the <iframe> embed code

4.7.3 Third: Add the Calendar to Your Site (optional)

Create a new file in your project called calendar.qmd and add the following. Replace the src URL with your own calendar’s embed URL:

---
title: "Event Calendar"
toc: true
page-layout: article
---

# Event Calendar

<div style="
  max-width: 1100px;
  margin: 0 auto 50px auto;
  border-radius: 22px;
  background: #ffffff;
  box-shadow: 0 12px 25px rgba(0,0,0,0.15);
  overflow: hidden;
">

  <iframe 
    src="YOUR-GOOGLE-CALENDAR-EMBED-URL-HERE"
    style="border:0; width:100%; height:900px; display:block;"
    frameborder="0"
    scrolling="no"
  ></iframe>

</div>

4.7.4 Fourth: Add the Calendar to Your Navigation Bar (optional)

Open _quarto.yml and add your new calendar page to the navbar under the left: section:

navbar:
  left:
    - href: calendar.qmd
      text: Calendar
    - href: research.qmd
      text: Research

Save _quarto.yml, then run quarto preview in the Terminal to see your calendar page live.

Note

The calendar will only display correctly when viewed in a browser — it won’t show up in RStudio’s built-in viewer panel. If you don’t see it in the RStudio preview pane, copy the local address (e.g. http://localhost:4321) and open it in Chrome or Firefox.

4.8 Step 7: Save and Preview Your Changes

After making any edits in index.qmd or _quarto.yml, save the file with Ctrl+S (Windows) or Cmd+S (Mac). If quarto preview is already running in the Terminal, your browser should refresh automatically within a few seconds.

If the preview doesn’t update, go to the Terminal and run:

quarto preview
Important

If you see an error in the Terminal after saving, read the message carefully — Quarto’s error messages usually tell you exactly which file and line the problem is on. The most common issues are mismatched ::: column dividers, incorrect image filenames, or indentation errors in _quarto.yml.

4.9.3 Suggested Workflow for Learning More

  1. Build your basic website using this tutorial
  2. Identify one thing you want to improve (layout, styling, navigation)
  3. Use the documentation above to find that specific feature
  4. Implement and test it
Important

Avoid trying to learn all of Quarto at once. Focus on small, incremental improvements.
